## 2. Client idéal

### Profil type
| Critère | Détail |
|---------|--------|
| **Taille flotte** | 5 à 50 véhicules (sweet spot : 10-30) |
| **Secteur** | Transport routier de marchandises, frigorifique, messagerie, BTP, services |
| **Rôle décideur** | Exploitant transport, directeur d'exploitation, gérant TPE/PME |
| **Équipe** | 1 à 5 personnes au bureau, 5 à 50 chauffeurs |
| **Outil actuel** | Excel, tableau blanc, classeurs papier, ou logiciel vieillissant |

### Douleurs quotidiennes
1. **Amendes DREAL** — CT oublié, visite médicale expirée, tachygraphe non étalonné → amende + immobilisation
2. **Pannes non anticipées** — Pas de suivi kilométrique des vidanges/courroies → panne sur route = perte de CA
3. **Temps administratif** — 2-3h/jour à chercher des infos dans des classeurs, relancer les garages, vérifier les documents
4. **Aucune visibilité** — Pas de tableau de bord, pas de score de fiabilité, pas d'alerte automatique
5. **Sinistralité** — Pas de traçabilité des incidents, pas de scoring conducteur

### Ce qu'il perd sans FleetMaster
- **1 500 à 5 000€/an** en amendes évitables (CT, tachygraphe, visite médicale)
- **3 à 10 jours d'immobilisation/an** par véhicule mal entretenu
- **10h/semaine** de travail administratif manuel
- **Aucune preuve** en cas de contrôle DREAL (pas de carnet d'entretien numérique)

## 7. Analyse technique — Mode offline / Zones blanches

### Ce que le code révèle

#### Service Worker (PWA) — PRÉSENT
**Fichier** : `public/sw.js`

FleetMaster Pro est une **Progressive Web App (PWA)** installable avec :
- `manifest.json` configuré en mode `standalone` (fonctionne comme une app native)
- Icônes PWA de 72x72 à 512x512 pixels
- Raccourcis rapides (Dashboard, Véhicules, SOS Garage)

#### Stratégie de cache — PARTIELLE

| Type de ressource | Stratégie | Offline ? |
|-------------------|-----------|-----------|
| JS/CSS (bundles Next.js) | **Cache-first** (permanent) | ✅ Oui |
| Icônes, images statiques | **Cache-first** (permanent) | ✅ Oui |
| Pages HTML | **Network-first** (pas de cache) | ❌ Non |
| Appels API (`/api/*`) | **Network-first** avec cache 1h | ⚠️ Lecture seule (données de la dernière heure) |
| Push notifications | Handlers dans le SW | ✅ Reçues même app fermée |

#### Formulaires conducteurs — NON FONCTIONNELS OFFLINE

Le code **ne contient pas** :
- ❌ Pas d'IndexedDB pour stocker les formulaires en local
- ❌ Pas de Background Sync API pour renvoyer les données quand la connexion revient
- ❌ Pas de file d'attente de mutations offline
- ❌ Pas de localStorage pour les saisies en cours

**Concrètement** : si un chauffeur remplit une inspection ou un plein de carburant en zone blanche, la soumission échouera. Les données ne seront pas sauvegardées localement ni synchronisées automatiquement.

#### Ce qui fonctionne en zone blanche
- ✅ L'application s'ouvre (shell PWA en cache)
- ✅ Les CSS/JS chargent (cache permanent)
- ✅ Les données consultées dans la dernière heure s'affichent (cache API 1h)
- ✅ Les push notifications reçues avant la perte de connexion restent visibles

#### Ce qui ne fonctionne PAS
- ❌ Aucune saisie de données (inspection, carburant, incident)
- ❌ Pas de consultation des données si le cache API a expiré (>1h)
- ❌ Pas de synchronisation automatique au retour du réseau
- ❌ Le module SOS Garage ne fonctionne pas (requiert un appel API)

### Faisabilité technique du mode offline complet

| Composant | Effort | Description |
|-----------|--------|-------------|
| **IndexedDB local** (Dexie.js) | 2-3 jours | Stocker véhicules, chauffeurs, dernières inspections en local |
| **File d'attente mutations** | 3-4 jours | Intercepter les POST/PUT quand offline, stocker en IndexedDB, rejouer au retour |
| **Background Sync API** | 1-2 jours | Déclencher automatiquement la synchronisation quand le réseau revient |
| **Indicateur offline** | 0.5 jour | Bandeau "Vous êtes hors connexion — vos données seront envoyées au retour du réseau" |
| **Gestion des conflits** | 2-3 jours | Résoudre les cas où les mêmes données ont été modifiées online et offline |
| **Tests & stabilisation** | 2-3 jours | Tests en conditions réelles (mode avion, réseau lent, 3G intermittent) |

**Estimation totale** : 10-15 jours de développement pour un mode offline complet et fiable.

**Alternative rapide** (3-5 jours) : Mode offline partiel — seuls les formulaires chauffeurs (inspection, carburant, incident) fonctionnent offline avec synchronisation basique au retour du réseau. Pas de consultation offline des données historiques.

## Annexe A — Fonctionnalités implémentées (vérifiées dans le code)

### Gestion de flotte
- Ajout/modification/suppression véhicules (VOITURE, FOURGON, POIDS_LOURD, POIDS_LOURD_FRIGO)
- Import CSV en masse
- Score de fiabilité IA par véhicule (maintenance 40% + inspections 35% + consommation 25%)
- QR Code par véhicule pour inspections rapides
- Carnet d'entretien numérique exportable en PDF (format DREAL)
- Suivi kilométrique

### Gestion chauffeurs
- Profils complets (permis, CQC, visite médicale, ADR, FCO/FIMO)
- Scoring sécurité par chauffeur
- Application mobile dédiée (inspections, carburant, incidents)
- Gestion des accès (création/révocation de compte chauffeur par l'admin)
- Suivi documents avec alertes d'expiration

### Maintenance
- Demandes avec workflow d'approbation (Directeur → RDV → En cours → Terminée)
- Types : préventive, corrective, pneumatique, carrosserie
- Priorités : basse, normale, haute, critique
- **Maintenance prédictive IA** — moteur de règles par type de véhicule (20+ catégories : moteur, freinage, transmission, filtration, réfrigération, etc.)
- Planning maintenance avec statuts : OK / À planifier / En retard
- Rappels automatiques J-1 par email

### Conformité réglementaire
- Suivi CT, tachygraphe, ATP, assurance, visite médicale, permis
- Score de conformité flotte (0-100%)
- Export PDF rapport DREAL
- Export CSV format DREAL pour chauffeurs
- Alertes automatiques 60j / 30j / 7j avant expiration
- Gestion par activité : marchandises, ADR, frigorifique, international, scolaire

### Module SOS Garage
- Diagnostic de panne en 4 étapes
- Recherche garages partenaires / externes
- Protocoles d'urgence personnalisables
- Contacts d'urgence autoroute / dépanneur

### IA intégrée
- **Assistant réglementaire** — Claude Haiku 4.5, streaming, questions sur le code de la route, 561/2006, ADR, tachygraphes
- **Briefing quotidien** — Résumé IA de l'état de la flotte
- **Analyse anomalies carburant** — Détection de consommation anormale
- **Scoring véhicules et chauffeurs** — Algorithme pondéré

### Carburant & coûts
- Saisie pleins (litres, coût, type carburant)
- Consommation L/100km par véhicule et chauffeur
- Détection d'anomalies (IQR)
- TCO (Total Cost of Ownership) sur 3/6/12/24 mois
- Export CSV

### Incidents & sinistralité
- Déclaration avec photos et géolocalisation
- Types : collision, mécanique, vol, incendie
- Suivi assurance
- Export PDF rapport d'incident

### Inspections
- Formulaire par QR code (sans authentification nécessaire)
- Photos de défauts
- Score d'inspection
- Historique par véhicule

### Notifications & alertes
- Push notifications (Web Push API + VAPID)
- Emails automatiques (Resend)
- Notifications in-app
- Préférences par utilisateur

### Rapports automatiques
- Rapport mensuel par email (flotte, conformité, carburant, maintenance)
- Rapport hebdomadaire
- Cron jobs : 12 tâches planifiées (maintenance, conformité, scoring, alertes, nettoyage)

### API publique v1
- 8 endpoints REST (véhicules, chauffeurs, carburant, maintenance, conformité, alertes)
- Documentation Swagger interactive (`/api-docs`)
- Rate limiting par plan (100 / 1 000 / 10 000 req/h)
- Authentification par clé API

### Agenda
- Vue calendrier (mois/semaine/jour)
- Événements : maintenance, CT, tachygraphe, ATP
- Code couleur par type

### Administration
- Gestion équipe (rôles : Admin, Directeur, Agent de parc, Exploitant, Chauffeur)
- Journal d'activité (audit trail)
- Super admin : gestion multi-tenant, analytics plateforme, support
